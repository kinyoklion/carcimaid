# Corpus attribution

The compliance corpus is organised by diagram type (e.g. `corpus/flowchart/`).
Each `.mmd` file is one test case. Provenance:

## Hand-written seed cases (carcimaid, MIT)
The seed cases currently in `corpus/flowchart/` (`simple-chain`, `decision`,
`shapes`, `lr-edge-styles`) are original, written for this project and covered
by the repository's MIT license. They exist to exercise the parser/layout/render
subset as it grows.

## mermaid-derived cases (MIT, under `*/mermaid/`)
The bulk of the corpus is diagram source harvested from the mermaid project,
which is MIT-licensed (`Copyright (c) 2014 - 2022 Knut Sveidqvist`). Cases live
in a `mermaid/` subdirectory per type (`corpus/<type>/mermaid/`) so their origin
is unambiguous. Coverage spans every mermaid diagram type carcimaid targets —
`flowchart`, `sequence`, `class`, `state`, `er`, `gantt`, `pie`, `journey`,
`git`, `mindmap`, `timeline`, `quadrant`, `requirement`, `c4`, `sankey`,
`xychart`, `block`, `packet`, `kanban`, `architecture`, `radar`, `treemap`,
`zenuml` — harvested against mermaid **11.15.0** (matching the oracle). Sources:

- `cypress/integration/rendering/**/*.spec.{js,ts}` — backtick template literals
  (interpolated `${...}` ones are skipped).
- `demos/*.html` — `<pre class="mermaid">` blocks.

Each diagram's type is detected from its header keyword and routed to the
matching `corpus/<type>/mermaid/` directory. The corpus is intentionally broad:
carcimaid renders only a subset of these types today and reaches greater
compatibility incrementally, so many cases are expected to fail compliance until
their renderer lands. `corpus/flowchart/` is hand-curated (some cases carry
restored `look:`/`theme:` config) and is **not** regenerated by the harvester.

The harvested `.mmd` files are kept **byte-faithful** to upstream (no injected
comments, so they round-trip through both renderers unchanged). Provenance for
every file — upstream path and block index — is recorded in the generated
`SOURCES.tsv` manifest alongside them. Regenerate the non-flowchart types with:

```sh
git clone --depth 1 --branch 'mermaid@11.15.0' \
  https://github.com/mermaid-js/mermaid.git
# Harvest everything except the curated flowchart corpus:
python3 tools/harvest_corpus.py mermaid /tmp/corpus_new
cp -r /tmp/corpus_new/{sequence,class,state,...} corpus/
```

The full MIT notice is reproduced in the repository root `ATTRIBUTION.md`.

> Note: mermaid does **not** publish reusable baseline images (its visual
> regression baselines live in a private Argos cloud account). We therefore do
> not redistribute any mermaid images — reference SVGs are generated locally by
> running the mermaid CLI oracle.
